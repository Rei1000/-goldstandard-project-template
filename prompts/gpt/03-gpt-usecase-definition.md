# 03 – GPT Use Case Definition

## Ziel

Dieser Prompt dient dazu, die fachlichen Use Cases des Projekts sauber zu identifizieren, zu strukturieren und zu priorisieren.

Er baut auf den Ergebnissen von `02-gpt-project-definition.md` auf.

---

## Anweisung an ChatGPT

Wir haben bereits Projektidee, Zielgruppe, Scope und Hauptfunktionen definiert.

Jetzt sollst du die relevanten Use Cases fachlich strukturieren.

WICHTIG:
- Keine technische Architektur
- Keine Implementierung
- Keine API- oder UI-Details
- Fokus auf Nutzer, Auslöser, gewünschtes Ergebnis und Priorität

---

## Deine Aufgabe

Arbeite die zentralen Use Cases des Projekts aus und priorisiere sie.

Für jeden Use Case sollst du klären:

1. Name des Use Cases
2. Primärnutzer
3. Auslöser / Trigger
4. Ziel / gewünschtes Ergebnis
5. Kurzbeschreibung
6. Priorität (hoch / mittel / niedrig)
7. Abgrenzung / was nicht dazugehört

---

## Regeln

- Keine technischen Lösungswege beschreiben
- Keine unnötige Granularität
- Nicht zu früh spezialisieren
- Use Cases fachlich und verständlich halten

---

## Erwarteter Output

Am Ende musst du eine strukturierte Übersicht liefern mit:

- Liste der priorisierten Use Cases
- Primärnutzer je Use Case
- Trigger
- gewünschtes Ergebnis
- Kurzbeschreibung
- Abgrenzung

---
